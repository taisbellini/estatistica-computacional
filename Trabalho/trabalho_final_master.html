Introdução

A criptografia consiste no estudo de algoritmos para encripitar e decriptar mensagens para que o conteúdo permaneça seguro e sigiloso entre o remetente e o destinatário. Chamamos de texto simples a mensagem original, texto cifrado a mensagem encriptada, cifra o algoritmo utilizado e chave a função utilizada para codificar e decodificar o texto. Existem diversos tipos de algoritmos para criptografar textos que variam o nível de segurança e complexidade e podem ser classificados de diferentes maneiras. Dentre os algoritmos clássicos, há o de substituição, que consistem em apenas substituir cada letra por outra, e transposição, que reordenam os caracteres do texto de acordo com uma função.(n.d.a) Pode-se ainda combinar as duas (cifra de produto (n.d.b)), ou utilizar estes métodos como base para criptografias mais complexas (n.d.a). O artigo escolhido (n.d.b) busca utilizar o algoritmo Metropolis [], uma versão dos algoritmos de aproximação Markov Chain Monte Carlo (MCMC), para decifrar mensagens encriptados de forma iterativa. Estes métodos já haviam sido utilizados previamente por (Diaconis 2008) e (n.d.c), porém apenas para cifras de substituição. (n.d.b) utiliza MCMC para decifrar também códigos que foram criados através do método de transposição e substituição-mais-transposição obtendo bons resultados para chaves de até 40 caracteres. Para a implementação do algoritmo, é usado um texto referência onde se computa uma matriz de frequência dos bigramas (sequências de duas letras) neste texto. Em seguida, para cada suposta chave de decodificação que transforma o texto cifrado em texto simples, salva-se a frequência de bigramas gerados por esta transformação e se compara os bigramas gerados com a matriz de referência para analisar a probabilidade de esta chave estar correta.
